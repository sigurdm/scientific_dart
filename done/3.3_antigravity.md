## Implementation Summary

### 1. Unmanaged C Kernels (`custom_sorting.c`, `custom_sorting.h`)
- Developed high-performance sorting and searching C math algorithms without using C stdlib `qsort`:
  - Custom QuickSort (unstable, median-of-three pivot selection).
  - Custom HeapSort (unstable, guaranteed $O(N \log N)$ worst-case complexity).
  - Custom multi-pivot recursive Quickselect (for partition and argpartition) capable of partitioning around arbitrary sequence of `kth` elements in-place.
  - Custom binary search insertion finder supporting `side` boundaries (left and right) and indirect searching via sorted indices (`sorter`).
- Templated all algorithms via generic C macros for type safety and extreme performance. Instantiated them across all 6 primitive types (`double`, `float`, `long long`, `int`, `complex128_t`, `complex64_t`).

### 2. FFI Interop & Routing (`ndarray_bindings.dart`)
- Exposed kind-parameterized sorting FFI signatures and new FFI functions for `partition`, `argpartition`, and `searchsorted` in `ndarray_bindings.dart`.

### 3. Public APIs (`operations.dart`)
- Injected new `SortKind` and `SearchSide` enums, perfectly mapping NumPy behaviors to typed Dart paradigms.
- Refactored `sort` and `argsort` to accept the new `SortKind kind` parameter, routing them cleanly to our custom native implementations.
- Implemented public `partition`, `argpartition`, and `searchsorted` APIs:
  - In `partition` and `argpartition`, implemented axis-swapping to leverage fast contiguous unmanaged heap operations.
  - In `searchsorted`, implemented shape matching (the return index array mirrors the arbitrary shape of `v` exactly as in NumPy).
  - Built robust boolean sorting and binary search fallback logic natively in Dart using list views.
